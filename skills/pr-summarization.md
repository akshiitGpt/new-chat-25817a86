# PR Summarization

Create clear GitHub PR descriptions for human review.

Every PR body must include:
- Summary of instrumentation changes.
- Source Linear ticket URL.
- Notion event spec URL.
- Generated events and typed properties.
- Files changed and implementation approach.
- Tests run and results, or no-test rationale.
- Reviewer notes and uncertainty.

Rules:
- Do not request auto-merge.
- Do not hide uncertainty.
- Do not include secrets, tokens, private keys, authorization headers, or secret-bearing URLs.
